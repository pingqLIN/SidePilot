## 1. 目前已完成進度 (Done)

我們主要針對「SDK Bridge」（也就是 `scripts/copilot-bridge` 裡的 Node.js 服務）進行了大幅升級，並且已通過 TypeScript (`tsc --noEmit`) 編譯驗證：

### 後端實作 (Bridge - `server.ts` & `session-manager.ts`)：

1. **WP-01 (`Permission API`)：**
   - 實作了非同步的 pending permissions 佇列。
   - 新增白名單機制 (`readTextFile`, `listDirectory` 自動放行)。
   - 新增 60 秒 Timeout 自動拒絕機制。
   - `server.ts` 已新增以下路由：
     - `GET /api/permissions`
     - `POST /api/permission/resolve`
     - `GET /api/permissions/whitelist`
     - `POST /api/permissions/whitelist`
     - `GET /api/permissions/stream` (SSE 推送)
2. **WP-02 (`Timeout API`)：**
   - 解決 CLI 卡死問題：導入 `AbortController` + `Promise.race()`。
   - 發生 Timeout 時（預設 120s），會強制清除殭屍程序 (`taskkill` 或 `SIGKILL`) 清理環境。
3. **WP-04 (`Config 解析與環境變數`)：**
   - 新增 Server 端環境變數解析 (`GITHUB_COPILOT_CLI_PATH`, `GITHUB_COPILOT_TIMEOUT` 等)。
   - `GET /api/config` 路由。
4. **WP-05 (`動態 Model 列表與快取`)：**
   - 改為以 `spawn` 呼叫 CLI 取得模型清單。
   - 加入 TTL (10分鐘) 快取機制以加速回應，若失敗則 Fallback 回預設硬編碼清單。
   - `GET /api/models/info` 路由。
5. **WP-07 (`Prompt Strategy 策略`)：**
   - 新增三種 Prompt 輸出擴展模式 (`normal`, `concise`, `one-sentence`) 及最大上下文歷史輪數切換。
   - 在 API `/api/chat` 發送 prompt 前自動掛載策略後綴要求。
   - `GET /api/prompt/strategy` 與 `POST /api/prompt/strategy`。

### 前端實作 (Extension - `sidepanel.html`)：

- 已在 HTML 的結尾注入 **WP-01 Permission 權限要求確認對話框 (`#permissionModal`)**。
- 已在 HTML 的「SDK 模式屬性設定 (`#settings-tab`)」內注入 **WP-07 Prompt 策略切換按鈕 (`#promptStrategyBtns`)**。
